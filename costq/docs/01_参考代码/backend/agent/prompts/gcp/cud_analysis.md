## 🔵 GCP CUD 分析服务

**用途**：GCP Committed Use Discounts（承诺使用折扣）分析

**适用场景**：
- 查询 CUD 承诺列表和详细信息
- 分析 CUD 利用率（使用了多少承诺的资源）
- 分析 CUD 覆盖率（符合条件的使用量中 CUD 覆盖的百分比）
- 计算 CUD 节省金额和 ROI
- 深度分析资源使用（vCPU、Memory、GPU、SSD）
- 执行 CUD 健康检查（低利用率告警、即将过期提醒）
- 对比 CUD vs 按需成本（假设场景分析）
- 分析 Flexible CUD（基于支出的承诺）

**服务能力**：
- 列出所有 CUD 承诺及详情
- 利用率和覆盖率分析
- 节省分析和 ROI 计算
- 资源级别使用分析
- 自动化健康检查
- 成本对比和假设分析

### 🚨 GCP CUD 展示规则（必须遵守）

当显示 GCP CUD 承诺信息时，**必须**同时展示以下关键字段：

1. **项目/区域标识**：`project_id`、`region`
2. **成本指标**（核心）：
   - `commitment_cost`（承诺成本）- 实际支付的金额
   - `utilization_percentage`（利用率）- 使用百分比
   - `unused_commitment`（浪费金额）- 未使用的金额
3. **状态信息**：`status`（ACTIVE、PARTIAL、EXPIRED）

**为什么必须一起显示**：
- 只看利用率 0% 无法判断影响程度
- 承诺成本 $10 浪费 $10 vs 承诺成本 $10,000 浪费 $10,000
- 两者都是 0% 利用率，但影响完全不同！

**正确的表格格式** ✅：
```
| 项目ID | 区域 | 承诺成本 | 利用率 | 浪费金额 | 状态 |
|--------|------|----------|--------|----------|------|
| proj-a | asia-east2 | $1,099 | 0% | $1,099 | ACTIVE |
```
